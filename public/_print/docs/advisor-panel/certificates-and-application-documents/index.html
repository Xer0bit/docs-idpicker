Certificates and Application Documents

How to issue achievement certificates and review student application document uploads

This guide covers two closely related features: issuing platform-generated certificates to recognise student achievements, and reviewing the application documents students upload as part of their university or programme applications.

1. Issuing a Certificate

Certificates are formal, advisor-generated recognitions that appear in the student’s Certificates module and can be downloaded as PDFs.

  1. In the sidebar, click Certificates.
  2. Click Issue Certificate.
  3. A certificate builder dialog opens. Fill in:
    • Student — Search and select the recipient.
    • Certificate Type — Choose from available templates (e.g., Assessment Completion, Programme Readiness, Participation).
    • Issue Date — Defaults to today; change if back-dating is required.
    • Notes (Optional) — Add a personalised message that will appear on the certificate document.
  4. Click Issue.
  5. The certificate is immediately added to the student’s account. They will see it in their Certificates and Documents section and receive a notification.

Note: Once a certificate is issued, it cannot be edited. If you made an error, you must revoke the certificate and reissue a corrected one.

2. Reviewing Issued Certificates

To keep track of what has been issued:

  1. On the Certificates page, you see a table of all certificates you have issued.
  2. Filter by Student Name or Certificate Type to narrow the list.
  3. Each row shows the issue date, student name, type, and current status (Active or Revoked).
  4. Click any row to open the certificate detail and preview the generated document.

3. Revoking a Certificate

If a certificate was issued by mistake or needs to be corrected:

  1. Find the certificate in your issued list.
  2. Click the action menu (...) next to it and select Revoke.
  3. Confirm the revocation prompt.
  4. The certificate is immediately marked as Revoked and disappears from the student’s active certificate list.

Warning: Revoking a certificate cannot be undone for audit trail purposes. If you need to correct an error, revoke the original and issue a new one.

4. Reviewing Student Application Documents

Students upload supporting documents (transcripts, passports, language certificates) as part of their application process. Your job is to verify, flag, or reject each uploaded file.

  1. In the sidebar, click Application Documents.
  2. The list shows all students who have active document collections, each with a completion progress bar.
  3. Click a student row to open their Document Collection.

5. Verifying Individual Documents

  1. Inside a student’s collection, you see a checklist of required document steps.
  2. Click on a specific step (e.g., High School Transcript).
  3. The uploaded file appears — click Preview or Download to inspect it.
  4. Set the review status:
    • Verified — The document is valid and meets requirements.
    • Pending — You need more time to verify, or you are waiting for additional context.
    • Invalid — The document is incorrect, expired, or does not meet requirements.
  5. If you mark the file Invalid, fill in the Reason field with a clear explanation so the student knows exactly what to fix.
  6. Click Save Review.

6. Completing a Collection

Once all individual document steps are verified:

  1. Return to the collection overview for the student.
  2. Confirm all steps show a green Verified badge.
  3. Click Approve Collection to formally mark the student’s application document set as complete.
  4. The student receives a notification that their documents have been approved.

Common Questions

Situation What to do
A student says they uploaded a file but I cannot see it Refresh the collection detail page — if it still does not appear, verify the student completed the upload step rather than just selecting the file
I need to leave a note for a student about a rejected document Use the Reason field when setting a step to Invalid. The student can read this note in their own document panel
I issued a certificate with the wrong date Revoke the original certificate immediately and re-issue a new one with the correct date
