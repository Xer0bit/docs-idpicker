Product Scope and Roles

IDPicker panel architecture, actor model, and account types

This section defines how IDPicker is structured across user roles and panel responsibilities.

Core Panels

  • Student Panel
  • Advisor Panel
  • Parent Panel
  • Admin Panel

Account Type Model (Student)

  • Linked Student: connected to parent/guardian and can have shared visibility for selected progress areas.
  • Individual Student: independent account with self-managed preferences and direct platform usage.

Role Summary

Role Primary Goal Key Capabilities
Student Career and university planning Assessments, recommendations, mentorship, resources
Advisor Student guidance and supervision Manage students, create sub-advisors, monitor outcomes
Parent Progress visibility and support Access linked student insights and support tools
Admin Platform governance and operations Manage users/orgs, permissions, prompts/translations, monitoring
